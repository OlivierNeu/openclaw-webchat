# Bridge Protocol

This document describes the browser-facing contract exposed by the OpenClaw
WebChat Bridge. Third-party frontends should depend on this contract, not on
OpenClaw Gateway frames directly.

## Versioning

Current protocol version: `0.1`.

The bridge exposes the current version through:

```http
GET /api/capabilities
```

Until `1.0`, message shapes can still evolve. Breaking changes must be recorded
in this document and covered by regression tests.

## HTTP Endpoints

### `GET /health`

Unauthenticated liveness probe.

Response:

```json
{"status": "ok"}
```

### `GET /api/capabilities`

Unauthenticated discovery endpoint for frontends.

Response includes:

- bridge name and version;
- protocol version;
- WebSocket path;
- supported browser messages;
- supported bridge messages;
- feature flags.

### `GET /api/me`

Authenticated Firebase identity check.

Header:

```http
Authorization: Bearer <firebase-id-token>
```

In development only, when `ALLOW_DEV_AUTH=true`, the token can be
`dev:<email>`.

### `GET /api/media/outbound/{filename}?scope={scope}&exp={expiresAt}&fp={fingerprint}&sig={signature}`

Signed download endpoint for files generated by OpenClaw.

Rules:

- `filename` must be a basename, not a path.
- `scope` is an opaque HMAC derived from the OpenClaw session key.
- `exp` is a Unix timestamp. Expired links are rejected.
- `fp` is the file fingerprint observed when the link was generated. If a future
  file reuses the same basename with different size or modification time, the old
  link is rejected.
- `sig` is an HMAC over the filename, scope, expiry and fingerprint, generated
  by the backend with `OPENCLAW_MEDIA_LINK_SECRET`.
- the backend reads files from `OPENCLAW_MEDIA_OUTBOUND_DIR`.

Generated Markdown links are relative by default. In split frontend/backend
deployments, set `OPENCLAW_WEBCHAT_PUBLIC_BASE_URL` so links become absolute and
resolve on the bridge origin.

Media links are bearer links for a short time window. Treat them as sensitive and
do not persist them as permanent document URLs.

## WebSocket Endpoint

```text
WS /ws/chats/{chatId}
```

`chatId` is a frontend-stable conversation identifier. The backend maps it to
an OpenClaw `sessionKey` based on the authenticated user configuration.

## Authentication

The WebSocket supports three auth forms:

1. Initial message after socket open:

```json
{"type": "auth", "idToken": "<firebase-id-token>"}
```

2. Query string:

```text
/ws/chats/{chatId}?idToken=<firebase-id-token>
```

3. Authorization header, for clients that can set headers:

```http
Authorization: Bearer <firebase-id-token>
```

The reference browser frontend uses the initial `auth` message because browser
WebSocket APIs do not support custom headers.

## Browser-To-Bridge Messages

### `auth`

```json
{"type": "auth", "idToken": "<firebase-id-token>"}
```

Required when no query token or Authorization header is provided.

### `ping`

```json
{"type": "ping"}
```

Response:

```json
{"type": "pong"}
```

### `chat.history`

```json
{"type": "chat.history", "limit": 200}
```

Requests an OpenClaw-backed history snapshot for the current session.

### `chat.send`

```json
{
  "type": "chat.send",
  "message": "Convert this file to PDF",
  "clientMessageId": "client-generated-id",
  "attachments": []
}
```

The bridge sends `sessions.patch` with `verboseLevel: "full"` before
`chat.send`, so tool results and `mediaUrls` are not stripped by OpenClaw.

### `chat.abort`

```json
{"type": "chat.abort"}
```

Requests cancellation of the current OpenClaw session run.

## Bridge-To-Browser Messages

### `bridge.ready`

Sent after authentication, user routing and OpenClaw connection succeed.

```json
{
  "type": "bridge.ready",
  "user": {"email": "user@example.com"},
  "target": {
    "instanceName": "olivier",
    "agentId": "olivier",
    "sessionKey": "agent:olivier:webchat:chat:..."
  }
}
```

### `chat.history`

Sanitized OpenClaw history payload.

```json
{"type": "chat.history", "payload": {"messages": []}}
```

### `chat.send.result`

OpenClaw acknowledgement for `chat.send`.

```json
{"type": "chat.send.result", "payload": {"ok": true}}
```

### `openclaw.frame`

Sanitized frame from OpenClaw for the current session only.

```json
{"type": "openclaw.frame", "frame": {"event": "chat", "payload": {}}}
```

Security rules:

- only frames whose `payload.sessionKey` matches the current session are
  forwarded;
- local `/home/node/.openclaw/...` paths are removed or converted;
- `mediaUrls` are converted to signed bridge URLs;
- OpenClaw credentials are never forwarded.

### `bridge.warning`

Non-fatal client usage issue, for example an unknown browser message.

### `bridge.error`

Fatal bridge or upstream error. The backend closes the WebSocket after sending
this message.

## Frontend Guidance

Third-party frontends should:

- call `/api/capabilities` before relying on optional features;
- use stable `chatId` values and persist them locally;
- reconcile state after reconnect by sending `chat.history`;
- treat `openclaw.frame` as sanitized but still untrusted display content;
- render Markdown links only with safe URL schemes;
- close the socket on sign-out.
